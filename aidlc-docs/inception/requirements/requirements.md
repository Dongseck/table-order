## Functional Requirements

### FR-01: 고객용 - 테이블 태블릿 자동 로그인

| ID | 요구사항 |
|----|----------|
| FR-01-1 | 관리자가 매장 식별자, 테이블 번호, 테이블 비밀번호를 1회 입력하여 초기 설정 |
| FR-01-2 | 로그인 정보를 로컬 스토리지에 저장 |
| FR-01-3 | 이후 접속 시 저장된 정보로 자동 로그인 수행 |
| FR-01-4 | 자동 로그인 실패 시 로그인 화면 표시 |

### FR-02: 고객용 - 메뉴 조회 및 탐색

| ID | 요구사항 |
|----|----------|
| FR-02-1 | 메뉴 화면이 기본(홈) 화면으로 항상 표시 |
| FR-02-2 | 카테고리별 메뉴 분류 및 표시 |
| FR-02-3 | 메뉴 상세 정보 표시: 메뉴명, 가격, 설명, 이미지 |
| FR-02-4 | 카테고리 간 빠른 이동 (탭/스크롤) |

| ID | 요구사항 |
|----|----------|
| FR-03-1 | 메뉴 추가/삭제 |
| FR-03-2 | 수량 조절 (증가/감소) |

| ID | 요구사항 |
|----|----------|

## Data Model Overview

### Core Entities
- **Store**: 매장 정보 (매장ID, 매장명, 설정)
- **AdminUser**: 관리자 계정 (매장에 소속)
- **Table**: 테이블 정보 (테이블번호, 비밀번호, 현재 세션ID)
- **TableSession**: 테이블 세션 (시작시각, 종료시각, 상태)
- **Category**: 메뉴 카테고리 (카테고리명, 정렬순서)
- **MenuItem**: 메뉴 항목 (메뉴명, 가격, 설명, 이미지URL, 카테고리, 정렬순서)
- **Order**: 주문 (주문번호, 테이블, 세션, 상태, 총금액, 주문시각)
- **OrderItem**: 주문 항목 (주문, 메뉴명, 수량, 단가)
- **OrderHistory**: 과거 주문 이력 (세션 종료 시 이동)

### Key Relationships
- Store → AdminUser (1:N)
- Store → Table (1:N)
- Table → TableSession (1:N, 현재 활성 세션 1개)
- TableSession → Order (1:N)
- Store → Category (1:N)
- Category → MenuItem (1:N)
- Order → OrderItem (1:N)
